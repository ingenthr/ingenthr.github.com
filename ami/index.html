There is no additional cost over that of running your own AMI. We would appreciate your feedback on the utility you receive from these AMIs. We have intentionally kept it simple, but if there is something you would like or need to be able to use these, please let us know.

To create these, we downloaded the latest stable release of memcached from memcached.org, built and tested memcached, and then integrated it's startup into the image. When the AMI boots, it will automatically use nearly all of it's memory for memcached.

In most cases, there is no reason to log into the image. Just launch it and point your client to port 11211 (either TCP or UDP). do be cautious about your security groups, as memcached does not have an authentication mechanism.
